# 异常处理 — 用户提示机制

在整个流程执行过程中，遇到以下情况时**必须暂停并提示用户**，不自行决定处理方式：

## 网络与 API 异常

| 异常类型 | 提示内容 |
|---|---|
| Tool 调用连续失败 ≥ 3 次 | `⚠️ 工具调用连续失败 <N> 次：<错误摘要>。建议检查网络或 API 状态后继续。` |
| API 返回 429/503/504 | `⚠️ API 服务暂不可用（<状态码>）。建议稍后重试或检查 API 配额。` |
| API 认证失败（403/401） | `⚠️ API 认证失败。请检查 token 或认证配置。` |

## 环境异常

| 异常类型 | 提示内容 |
|---|---|
| 构建命令执行失败 ≥ 2 次 | `⚠️ 构建连续失败。请检查构建环境配置，或手动验证后告知我继续。` |
| 应用启动超时或失败 | `⚠️ 应用启动异常。请检查启动命令或环境状态，手动确认后告知我继续。` |
| 数据库连接失败 | `⚠️ 数据库连接失败。请检查数据库状态和连接配置。` |

**处理原则：** 遇到异常后停止、报告、等待用户指示。不在用户不知情的情况下自动重试或绕过问题。
